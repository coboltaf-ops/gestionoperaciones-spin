6 EJECUCION AUTOMATICA

El sistema ejecuta automaticamente:

  • Crear el proyecto en Vercel
  • Configurar el subdominio (ej: crmgtmcol.sistech.com)
  • Crear la base de datos en Supabase
  • Instalar el sistema con los modulos seleccionados
  • Crear los usuarios con sus roles y permisos
  • Personalizar logo, nombre de empresa, ciudad y pais
  • Preparar configuracion base de correo (ver Paso 7 para personalizacion)

7 CONFIGURACION DE CORREO (IDENTIDAD DEL CLIENTE)

Configurar el servidor de correo para que los emails salgan con la identidad de la empresa del cliente. SIN ESTE PASO, los correos salen con remitente generico y los destinatarios no saben quien les envio.

Datos de Correo del Cliente:

GTM Grupo Empresarial S.A.S.
sistema@gtmgrupo.com
smtp.gmail.com / smtp del cliente
465 (SSL) o 587 (TLS)
sistema@gtmgrupo.com
********
  • Opcion A - Gmail: El cliente crea una cuenta Gmail (ej: notificaciones.gtm@gmail.com), genera una App Password, y la proporciona
  • Opcion B - Correo corporativo: El cliente proporciona datos SMTP de su dominio (ej: sistema@gtmgrupo.com)
  • El nombre de empresa aparecera en: remitente del email, header del PDF adjunto, header del email HTML, y footer del correo
  • Estos datos se configuran en: Variables de entorno de Vercel (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS) y en el modulo Datos Empresa del sistema
  • VERIFICAR: Enviar correo de prueba y confirmar que el destinatario ve el nombre de la empresa del cliente como remitente

Ejemplo de como se ve el correo recibido

De: GTM Grupo Empresarial S.A.S. <sistema@gtmgrupo.com>

Asunto: Orden de Compra OC-00015 - GTM Grupo Empresarial S.A.S.

El proveedor/empleado ve claramente que el correo viene de su cliente, no de un sistema generico.
