3 instancias independientes del CRM GTM por pais

13 de Abril de 2026
1. RESUMEN EJECUTIVO

Implementacion de 3 instancias independientes del sistema CRM GTM para el cliente TAMOIN, una por cada pais de operacion (Colombia, Ecuador, Peru). Cada instancia es completamente autonoma con sus propios datos, usuarios, correo y personalizacion.

Por que 3 instancias separadas?

  • Datos de empresa distintos (NIT, RUC, direccion, logo)
  • Correo SMTP propio — emails salen con la identidad de cada pais
  • Usuarios y permisos independientes por pais
  • Datos aislados — Colombia no ve datos de Ecuador ni Peru
  • Dashboard muestra el nombre del pais correspondiente

4. PROCESO DE IMPLEMENTACION

Los siguientes pasos se repiten para cada pais. El agente (Claude) ejecuta todo el trabajo tecnico.

1 RECOPILAR DATOS DEL CLIENTE

Solicitar al cliente la siguiente informacion por cada pais:

Datos Empresa (por pais):

Ej: Grupo GTM Colombia S.A.S.
Ej: 900.123.456-7 (COL) / 1790012345001 (ECU) / 20123456789 (PER)
Direccion de la oficina en ese pais
Bogota / Quito / Lima
Colombia / Ecuador / Peru
Telefono de contacto del pais
correo@empresa.com
Archivo PNG o JPG (puede ser el mismo o diferente por pais)

2 RECOPILAR USUARIOS INICIALES

Por cada pais, el cliente indica los usuarios que tendran acceso al sistema:

Por cada usuario:

Ej: Carlos Mendoza Rios
Ej: cmendoza
Ej: cmendoza@tamoin.com.co
Ej: Admin01*
Admin / Comercial / Operaciones / Contabilidad

Nota sobre permisos

Los usuarios Admin tienen acceso total. Para otros roles se configura la Matriz de Permisos por modulo (Leer, Registrar, Editar, Eliminar). Despues de la entrega, el Admin del cliente puede crear mas usuarios y modificar permisos sin nuestra ayuda.

3 RECOPILAR DATOS SMTP

Para que los correos del sistema salgan con la identidad de cada pais:

Opcion A — Gmail (recomendada, gratis):

Ej: notificaciones.tamoin.col@gmail.com
Se genera desde la configuracion de seguridad de Gmail

Opcion B — Correo corporativo:

Ej: smtp.tamoin.com.co
465 (SSL) o 587 (TLS)
Ej: sistema@tamoin.com.co
********

4 DEPLOY E IMPLEMENTACION (lo hace el agente)

Con los datos recopilados, el agente ejecuta automaticamente:

  • Clonar el proyecto CRM GTM base
  • Crear proyecto en Vercel (crmtamoincol / crmtamoinecu / crmtamoinper)
  • Configurar subdominio en sistech.com
  • Crear proyecto en Supabase
  • Configurar variables de entorno (SMTP, API keys)
  • Personalizar: logo, nombre empresa, NIT/RUC, ciudad, pais
  • Crear usuarios iniciales con roles y permisos
  • Activar/desactivar modulos segun lo contratado
  • Prueba completa del sistema
  • Envio de correo de prueba para verificar identidad

5 ENTREGA DE ACCESOS

Se genera documento de entrega con:

  • URL del sistema por pais
  • Usuarios y claves iniciales
  • Manual de usuario en PDF
  • Datos de contacto para soporte

6 CAPACITACION (4 sesiones)

  • Sesion 1: Navegacion general + Dashboard + Clientes + Contactos
  • Sesion 2: Oportunidades + Cotizaciones + Productos
  • Sesion 3: Proyectos + Tareas + Tickets + PQRS
  • Sesion 4: Correos + Reportes + Configuracion + Gestion de Usuarios
